Demo Cases
AI归因case示例 点击可查看

以下案例来自本地 Demo 归因样例，用于展示 AI 不是直接贴标签，而是从用户交互、促销信息和系统归因信息中逐步判断最终原因。

Case 1 · 用户误解
用户把“免费升房权益”纳入价格感知
输入 · 原始沟通
用户原声：“还可以免费升房”“同等价格可以免费升级，不能升级的还便宜”。客服核实：去哪儿到手价 125 元，同程到手价 134 元。
Agent 判断
IM 归因先识别客服判断为“价格不 lose”，再从对话里抽取“免费升房/免费升级”这类权益关注点，命中用户误解中的免费权益场景。
输出
不是价格或促销导致的 lose，而是用户把免费升级权益纳入价格比较，最终进入用户误解分支。
最终归因：用户误解 / 免费类权益lose
Case 2 · 促销 lose
竞品平台补贴更强，导致到手价低
输入 · 比价信息
客服判断为“可对比，lose，同质化 lose”。同程划线价/到手价为 459/421；去哪儿同质化房型划线价/到手价为 459/432。竞品促销包含订房享好价、门店新客、红包立减；去哪儿促销为出行特惠、房费立减。
Agent 判断
同程归因规则先判断划线价一致：|459 - 459| ≤ 0.1，所以不进入“划线价不一致”分支，继续比较促销分桶。Q 侧商促 23 元、平台促销 4 元；E 侧商促 16 元、普通平台券 22 元。
输出
商促桶 E 比 Q 少 7 元，不命中商家补贴；平台补贴桶 E 比 Q 多 18 元，抵消商促差异后形成到手价差。四级继续看平台补贴构成：E 普通平台券 22 元，Q 侧平券 0 元，命中“平券lose”。
最终归因：价格lose / 促销lose / 平台补贴lose / 平券lose
Case 3 · 划线价不一致
当前只能判断不是同一套货，三级原因待系统数据区分
输入 · 比价信息
客服判断为“可对比，lose，同质化 lose”。同程划线价/到手价为 955.55/880.55；去哪儿同质化房型划线价/到手价为 1393/1105。
Agent 判断
当前规则先判断划线价一致性：|1393 - 955.55| > 0.1，可以确认 Q 和 E 不是同一套货源/报价口径，不能继续用促销分桶解释价差。
输出
当前 Demo 只能输出到“划线价不一致”。后续三级需要依赖系统数据继续区分：轮转 lose、底价 lose（E 独有货源）、会员兑 lose & 活包资源 lose；这些原因当前不能由 AI 单独判断。
当前可判断：价格lose / 划线价不一致；三级原因待系统数据输入后再细分
